QUESTION 1: GAP (TRENCH) CALCULATION METHOD
============================================

Implementation Strategy:
------------------------
I used a ridge boundary-based projection approach:

1. Compute a 1D vertical projection (summing pixel intensities column-wise)
   to collapse the 2D texture.
2. Isolate ridge regions using a relative peak threshold (25% of the maximum
   projection intensity).
3. Calculate the physical gap dynamically: gap = next_ridge_start - current_ridge_end

Code (Cell 12 in notebook):
```python
gaps = []
for i in range(len(ridge_regions) - 1):
    gap = ridge_regions[i+1]['start'] - ridge_regions[i]['end']
    gaps.append(gap)
```

Why this method works well for SEM:
------------------------------------
- Noise Tolerance: Vertical projection naturally smooths over the internal
  granular texture (secondary electron noise) on the ridge surfaces, making
  it more robust compared to pure 2D Canny/Sobel edge detection.

- Accurate Trench Measurement: Unlike a peak-to-peak distance calculation
  (which only yields the pitch/period), this boundary method accurately
  isolates the empty trench space.


================================================================================
QUESTION 2: ALGORITHM ASSUMPTIONS & ROBUSTNESS
================================================================================

Current Pipeline Constraints & Assumptions:
--------------------------------------------
During robustness testing, I identified two primary assumptions:

1. FIXED SEARCH WINDOWS (Scale Dependency)
   - Assumption: The vertical projection relies on a fixed 150px search range,
     assuming relatively constant magnification/scale.
   - Limitation: The algorithm fails to segment properly if the image scale
     is doubled (2x zoom). Attempting to convert this to a dynamic percentage
     (e.g., image_height * 0.30) caused a cascade effect, requiring downstream
     threshold retuning.

2. UNIFORM BACKGROUND ILLUMINATION
   - Assumption: The dark substrate background is relatively uniform.
   - Limitation: Deep black padding or severe electron beam charging (bright
     spots) skews the global Histogram Equalization.
   - Production Fix: Implement masked equalization or localized CLAHE to
     normalize contrast only within valid structural regions.


Handling of Local Variations:
------------------------------
Traditional CV pipelines require hardcoded tolerances. For example:

- 15% Center of Mass (COM) offset: Required to capture asymmetric ridge peaks.
  Works for this image but might fail on symmetric ridges.

- Local threshold variation: Localized illumination variations (such as the
  darker S2 ridge) required manual threshold tuning (intensity 127 vs. 122).
  This is a known limitation of global thresholding. Production systems would
  use adaptive thresholding (CLAHE).

- Detection thresholds: 25% for projection, 0.7 for merging, 4% gap tolerance.
  All tuned by trial and error for this specific image.


Things My Algorithm Does NOT Assume:
-------------------------------------
My algorithm handles several cases well:

- Substrate position: Auto-detects using gradient detection + RANSAC
  (works anywhere in image, tested at 8° rotation)

- Brightness/contrast: Histogram equalization + Otsu thresholding adapts
  automatically to each image

- Scale bar: Has manual override fallback if automatic OCR fails


Robustness Testing (Augmentation Suite):
-----------------------------------------
To simulate real-world variations (stage tilt, defocus, electron beam noise),
I tested the pipeline against 7 augmented edge cases:

PASS (6/7):
- Gaussian Blur (defocus)
- Brightness shift (-40%)
- Gaussian noise (static)
- Rotation (8° stage tilt)
- Horizontal flip
- Position shift (60px up)

Note: The substrate baseline detection using RANSAC + gradient detection
proved highly resilient to the 8° rotation.

FAIL (1/7):
- 2x Magnification Scale (only detected 4/5 ridges due to the fixed pixel
  search window)


Architectural Trade-offs: Traditional CV vs. Deep Learning
-----------------------------------------------------------
This prototype successfully generalizes to 6 out of 7 targeted edge-case
augmentations without requiring any training data. However, the pipeline
relies on multiple interdependent hardcoded parameters (e.g., 25% projection
threshold, 0.7 merge ratio, 4% gap tolerance). Changing one parameter causes
a downstream cascade effect.

While this deterministic approach is excellent for a zero-data baseline,
production systems would benefit from transitioning to a Deep Learning
segmentation architecture (e.g., U-Net). A CNN would learn these spatial
hierarchies and illumination patterns from a dataset of 50-100 annotated
SEM images, eliminating brittle thresholds and scaling seamlessly across
different magnifications.


Summary:
--------
The algorithm successfully detects all 5 ridges on the original image with
accurate measurements. Robustness testing shows the pipeline generalizes to
6 out of 7 edge-case augmentations (detection only - measurement accuracy
on transformed images was not validated).

Primary limitation: Fixed 150px search range fails at different magnifications,
revealing the scale-dependent nature of traditional CV approaches.
